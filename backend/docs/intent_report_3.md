# intent_report.md  

## 1. 專案概述  
- **系統名稱**：小學二年級英語單詞學習系統  
- **主要目的**：透過遊戲化的互動情境（點擊、聲音、獎勵小物、積分機制）提升二年級學生學習英語單詞的動力、興趣與記憶成效。  

## 2. 商業意圖 (5 Whys)  

| Why 層級 | 問題 | 用戶答案 / 背後原因 |
|----------|------|--------------------|
| 1️⃣ | 想開發什麼系統？ | 給小學二年級學生學習英語單詞的系統。 |
| 2️⃣ | 為什麼要這樣的系統？ | 希望提升他們對英語的學習動力與興趣。 |
| 3️⃣ | 為什麼提升動力與興趣對您或組織重要？ | 想以遊戲互動的情境教學，讓孩子「邊玩邊學」並有效背單詞。 |
| 4️⃣ | 為什麼相信遊戲互動方式能提升動力與興趣？ | 傳統英語教學太死板，互動點擊、獎勵小物或積分可讓學習變得更有趣、更具挑戰性。 |
| 5️⃣ | 為什麼點擊、獎勵、積分機制能真正提升學習成效與長期記憶？ | 人對聲音與即時互動有天然的記憶加強效果，這類刺激能提升注意力、強化記憶痕跡，進而提升學習成效。 |

**核心商業價值**  
- **差異化**：以遊戲化、即時回饋與聲音交互創造比傳統教材更高的黏著度。  
- **市場需求**：家長與學校尋求能激發幼兒學習興趣的數位工具。  
- **長期效益**：提升英語基礎能力，為未來學習奠定基礎，並提升產品在教育軟體市場的競爭力。  

## 3. 功能需求 (從意圖抽取)  

| 功能類別 | 具體需求 | 目的說明 |
|----------|----------|----------|
| **單詞學習** | - 顯示二年級常見英語單詞（文字 + 圖示）<br>- 發音播放（聲音） | 提供基礎認知與聽說結合。 |
| **遊戲化互動** | - 點擊式配對/選擇遊戲<br>- 拖曳或點擊觸發動畫<br>- 成功後即時音效回饋 | 透過即時互動提升注意力與記憶。 |
| **獎勵機制** | - 收集「小物」或貼紙<br>- 積分系統（加分、排行榜）<br>- 解鎖新關卡/新主題 | 形成正向回饋迴路，增強動機。 |
| **進度與成就** | - 個人學習紀錄（已學、未學、正確率）<br>- 成就徽章展示 | 讓家長/教師可監控學習成效，學生可自我激勵。 |
| **管理後台** | - 單詞庫上傳與分類<br>- 遊戲情境/關卡設定<br>- 獎勵物件與積分規則設定 | 方便內容更新與客製化。 |
| **多平台支援** | - 網頁 (PC/Tablet) & 手機 (iOS/Android) | 符合學校與家庭使用情境。 |
| **安全與隱私** | - 無個人資料收集或僅限家長同意的最小化資料 | 符合兒童線上安全規範（如 COPPA、GDPR‑Kids）。 |

## 4. 非功能需求  

| 項目 | 需求說明 |
|------|----------|
| **使用者體驗** | UI 必須色彩鮮明、操作簡潔，適合 7–8 歲兒童。 |
| **響應速度** | 互動回饋（點擊→音效）延遲 ≤ 200 ms，以免打斷沉浸感。 |
| **可擴充性** | 後台可快速加入新單詞或新遊戲關卡。 |
| **可維護性** | 代碼遵循 modular/component 架構，便於未來功能迭代。 |
| **兼容性** | 支援主流瀏覽器 Chrome、Safari、Edge；手機支援 iOS 12+、Android 8+. |
| **可訪問性** | 提供文字替代說明與調整音量的控制，照顧聽力較弱的孩子。 |

## 5. 成功指標 (KPIs)  

| KPI | 目標值 | 測量方式 |
|-----|--------|----------|
| **使用時長** | 平均每位學生每日 ≥ 15 分鐘 | 後台使用紀錄 |
| **單詞掌握率** | 完成所有單詞正確率 ≥ 85%（測驗） | 系統測驗結果 |
| **活躍用戶** | 1‑月活躍率 ≥ 70% | 註冊帳號的日活/月活比 |
| **家長/教師滿意度** | NPS ≥ 30 | 調查問卷 |
| **留存率** | 第 30 天留存 ≥ 50% | 用戶留存曲線 |
| **獎勵兌換率** | ≥ 60% 的積分被使用於解鎖新內容 | 後台兌換紀錄 |

## 6. 利害關係人  

| 角色 | 需求/期望 | 主要關注點 |
|------|----------|------------|
| **小學二年級學生** | 好玩、易懂、能快速得到回饋 | 遊戲流暢度、聲音與視覺刺激 |
| **家長** | 孩子學習效果、使用安全、可監控 | 成績報告、隱私保護、無廣告 |
| **教師** | 補充課堂教學、快速批改、分組管理 | 數據分析、教材對齊課綱 |
| **產品經理/投資方** | 市場差異化、商業化、可持續營收 | 成本控制、付費功能（如進階關卡） |
| **開發團隊** | 明確需求、可重用元件、技術可行性 | 技術選型、開發時程 |

## 7. 限制與假設  

| 限制 | 說明 |
|------|------|
| **預算** | 先行開發 MVP（核心單詞 + 基本遊戲互動），後續再擴充關卡與獎勵。 |
| **時間** | 初版 3 個月交付（需求、設計、開發、測試）。 |
| **法規** | 必須符合當地兒童線上保護法規，無未經同意的資料蒐集。 |
| **技術** | 前端以 React (或 Vue) + Canvas/HTML5 遊戲引擎；後端以 Node.js + RESTful API。 |
| **內容** | 單詞庫由教育顧問提供，符合國小二年級教學大綱。 |

## 8. 建議的開發路線  

1. **需求驗證 (2 週)**  
   - 與教師、家長進行原型測試，確認遊戲化機制的吸引度。  

2. **MVP 設計 & 原型 (3 週)**  
   - 核心單詞 (≈ 30 個) + 基本點擊配對遊戲 + 簡易積分與獎勵系統。  

3. **開發迭代 (6 週)**  
   - 前端開發（UI、遊戲互動、音效）  
   - 後端 API（單詞、成績、積分）  
   - 後台管理介面（單詞上傳、關卡設定）  

4. **測試與部屬 (2 週)**  
   - 功能測試、兒童使用性測試、隱私合規檢查。  

5. **上市與回饋收集 (持續)**  
   - 收集 KPI、舉辦小學示範課程，迭代功能。  

---

**結論**  
透過 5 Whys 我們確認：客戶的核心痛點是「傳統英語教學缺乏互動與趣味，導致二年級學生學習動機不足」，解決方案應聚焦於 **遊戲化互動 + 聲音回饋 + 獎勵機制**，以提升記憶與學習成效。此 intent_report 為後續需求細化、設計與開發提供了完整且結構化的藍圖。
