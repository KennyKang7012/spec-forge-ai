# Proposal.md  

## 1. 项目概述  
为满足用户在「生活‑工作‑家庭‑学习」四大领域统一管理任务的需求，打造一款跨平台（iOS、Android、Web）记事工具。核心通过 **标签、紧急程度、里程碑提醒** 以及 **成就仪表板**，帮助用户可视化任务、提前预警并提升成就感。

---

## 2. MVP 范围（1‑3 个月）  

| 功能编号 | 功能名称 | 说明 | 是否纳入 MVP |
|----------|----------|------|--------------|
| **FR‑1** | 多域笔记 | 支持生活、工作、家庭、学习四大分类，可自行新增/编辑分类 | ✅ |
| **FR‑2** | 标签系统 | 用户自定义文字/颜色标签，支持多标签并列 | ✅ |
| **FR‑3** | 紧急等级 | 预设红、橙、黄、绿四级，可自定义；直接在笔记上设定 | ✅ |
| **FR‑5** | 即时提醒 | 根据紧急等级与里程碑时间，推送本地通知、邮件或推播 | ✅ |
| **FR‑8** | 跨平台同步 | iOS、Android、Web 三端数据实时同步 | ✅ |
| **FR‑4** | 里程碑与阶段 **（仪表板展示）** | 为笔记/任务设置里程碑，关联日期或完成条件；在仪表板中以图示方式呈现进度 | ✅（里程碑功能 + 仪表板展示） |
| **FR‑6** | 过滤/搜索 | 按标签、紧急等级、日期、关键字多维度过滤搜索 | ✅（基础实现） |
| **FR‑7** | 成就仪表板 | 显示已完成里程碑、任务完成率、近期高紧急任务统计等，可视化成就感 | ✅（与里程碑一起提供） |
| **FR‑9** | 数据导入/导出 | 支持 CSV、JSON、Markdown 导入导出 | ❌（留至第 4‑5 个月） |
| **FR‑10** | 协作功能 | 共享笔记、权限管理 | ❌（留至第 7‑8 个月） |

> **备注**：MVP 将聚焦核心任务管理与成就感展示，确保用户能在单一应用完成日常记录、优先级标记、提醒以及里程碑进度查看。

## 3. 非功能需求（全部阶段适用）

| 编号 | 要求 | 指标 |
|------|------|------|
| **NFR‑1** | 安全性 | TLS 1.3 传输，AES‑256 存储加密（符合 GDPR / 台灣個資法） |
| **NFR‑2** | 可用性 | 年可用率 ≥ 99.5% |
| **NFR‑3** | 响应时间 | 关键操作 < 300 ms |
| **NFR‑4** | 可扩展性 | 微服务/插件化架构，后续可加入 AI 摘要、语音输入 |
| **NFR‑5** | 跨平台一致性 | UI/UX 在 iOS、Android、Web 保持统一，UI 测试覆盖 ≥ 80% |
| **NFR‑6** | 离线使用 | 本地 SQLite 支持新增/编辑，网络恢复后自动同步 |

## 4. 技术选型  

| 层面 | 选型 | 理由 |
|------|------|------|
| **前端框架** | **React Native**（iOS & Android）+ **React**（Web） | - 单一代码库实现跨平台<br>- 丰富的生态与社区插件<br>- 易于集成本地 SQLite 与 Supabase SDK |
| **后端同步 & 数据库** | **Supabase**（PostgreSQL + Edge Functions） | - 即时同步（Realtime）支持<br>- 内建身份认证、Row‑Level Security 符合安全需求<br>- 与 PostgreSQL 完全兼容，便于后期分析报表 |
| **本地离线存储** | **SQLite**（通过 expo‑sqlite / react‑native‑sqlite‑2） | - 轻量、成熟、跨平台（iOS/Android/Web via IndexedDB）<br>- 与 Supabase Edge Functions 配合实现冲突解决与同步 |
| **即时提醒** | **Firebase Cloud Messaging**（推播） + **邮件服务**（SendGrid via Supabase Edge Function） | - 多渠道提醒确保高可靠性 |
| **身份认证** | Supabase Auth (OAuth + Email/Password) | - 与数据库权限天然集成，满足 NFR‑1 |
| **CI/CD** | GitHub Actions + Supabase Deploy | 自动化测试、部署与监控 |
| **监控 & 日志** | Supabase Logs + Sentry (前端) | 快速定位性能或异常问题 |

## 7. 风险与缓解  

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 需求蔓延 | 开发延期、资源紧张 | MVP 采用 **5 Whys** 验证，后期需求滚动至迭代计划 |
| 跨平台同步复杂度 | 数据不一致、用户流失 | 使用 Supabase Realtime + 本地 SQLite 双写模型，配合自动化集成测试 |
| 提醒失效 | 失去关键提醒，降低信任 | 多渠道（推播、邮件、本地通知） + 健康检查（心跳） |
| 合规安全 | 法规风险、用户流失 | TLS 1.3、AES‑256、定期安全审计、透明隐私政策 |
| 学习成本 | 低采纳率 | UI 采用「最小化操作」原则，提供交互式上手教程与模板 |

## 8. 结论  

本提案基于业务分析师提供的 **意图报告**，明确了用户对统一记事平台的根本需求。通过 **React Native + Supabase + SQLite** 的技术栈，实现跨平台、离线优先、即时提醒以及里程碑/成就可视化，为用户提供高效、可控且具成就感的任务管理体验。  

MVP 将在 3 个月内交付核心功能，让用户立即感受任务可视化与提前预警的价值；随后迭代加入协作、AI 与语音输入等高级特性，以实现长期增长与用户黏着。
