# Proposal: 每日工作日誌 & 研發知識庫週報系統  

**專案代號**：WeeklyLog‑KB  
**版本**：v1.0 (MVP)  
**提交日期**：2026‑05‑10  
**負責人**：產品經理 (PM)

---  

## 1. 商業背景與問題聲明  

| 項目 | 說明 |
|------|------|
| **核心需求** | 為全體員工提供一個「每日工作日誌」系統，記錄工作內容、工時與附件，並自動匯入研發重要資訊至公司知識庫。 |
| **痛點** | 1. 工作資訊與研發要點散落，缺乏統一、可搜尋的沉澱。<br>2. 主管與高層無法即時掌握團隊進度與決策依據。<br>3. 專案透明度不足，導致進度、風險與品質難以最佳化。 |
| **驅動因素** | 透過資訊集中、即時回饋與知識沉澱，提升專案交付品質、加速新產品上市、增進員工參與與營收共享。 |

---

## 2. 商業目標 & KPI  

| 目標 | 描述 | KPI |
|------|------|-----|
| **提升專案交付品質與效率** | 減少資訊斷層與重工 | - 專案延期率 ↓ 20% <br>- 缺陷密度 ↓ 15% |
| **建立可重用的知識庫** | 研發重要事項自動歸納 | - 知識庫內容增長 ≥ 30%/年 <br>- 搜尋命中率 ≥ 80% |
| **增強營收貢獻** | 加速產品規劃與上市 | - 新產品上市週期縮短 10% <br>- 營收增長率提升 5% |
| **提升員工參與與激勵** | 透過分紅機制提升歸屬感 | - 員工滿意度 ↑ 0.5/5 <br>- 離職率 ↓ 10% |

---

## 3. MVP 功能範圍  

| 功能編號 | 功能描述 | 重要度 |
|----------|----------|--------|
| **F1** | 員工每日工作日誌錄入（文字、附件、工時） | 高 |
| **F2** | 自動週匯總生成（依部門、專案） | 高 |
| **F3** | 主管/老闆檢視、批註、討論工作日誌 | 高 |
| **F4** | 研發重要事項標籤與分類（技術決策、問題、解法） | 中 |
| **F7** | 通知機制（新日誌、主管回覆、知識庫更新） | 中 |
| **F9** | 權限管理（員工、主管、管理層、系統管理員） | 高 |

> 以上六項為 **MVP**，後續版本將逐步加入 F5、F6、F8、F10 等功能。

## 4. 技術選型  

| 層級 | 技術 | 理由 |
|------|------|------|
| **前端** | **React** (Create‑React‑App + TypeScript) | 市場成熟、元件化開發、與企業內部 UI 標準相容 |
| **後端** | **Node.js / Express** | 高 I/O 效能、與 React 生態相輔、快速迭代 |
| **資料庫** | **MongoDB** | 文件型結構自然映射工作日誌與附件的 JSON 格式，彈性高 |
| **全文搜尋** | **ElasticSearch**（獨立服務） | 支援每日日誌與知識庫的即時全文檢索，滿足 ≤1 s 搜尋回應需求 |
| **認證/授權** | JWT + RBAC（基於 MongoDB 角色） | 符合最小權限原則，配合 TLS 1.3 加密 |
| **部署** | Docker + Kubernetes (藍綠部署) | 微服務化、彈性擴展、零停機升級 |
| **CI/CD** | GitHub Actions → Docker Registry → AKS/EKS | 自動化測試、建置與發布 |

## 5. 非功能需求  

| 項目 | 具體要求 |
|------|----------|
| **可用性** | Web (Chrome/Edge) + 行動裝置 Responsive；錄入流程 ≤ 3 分鐘 |
| **效能** | 同時 200 使用者上線，頁面載入 ≤ 2 秒；搜尋回應 ≤ 1 秒 |
| **安全** | TLS 1.3；資料分級存取；審計日誌；JWT + RBAC；定期安全掃描 |
| **可擴展性** | 微服務架構、支援未來多語系與跨部門使用 |
| **維運性** | 日誌資料備份 7 天；藍綠部署不影響服務 |
| **合規** | 符合公司資訊安全政策與 GDPR（如適用） |

## 6. 風險與緩解措施  

| 風險 | 影響 | 緩解策略 |
|------|------|----------|
| 使用者接受度低 | 系統不被日常使用，知識庫無法累積 | 內部培訓、使用指引、將日誌完成度與績效 KPI 連結 |
| 資料品質不佳 | 知識庫價值下降 | 強制必填欄位、定期自動提醒與審核 |
| 安全漏洞 | 機密資訊外洩 | 定期安全掃描、最小權限原則、TLS 1.3 |
| 系統整合失敗 | 與 HR／專案系統斷層 | 事前需求對接、使用標準化 REST API、提供測試環境 |
| 效能瓶頸 (搜尋慢) | 使用者體驗下降 | ElasticSearch 集群、讀寫分離、Cache 層 (Redis) |
