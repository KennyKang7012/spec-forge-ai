# 小學二年級英語單詞學習系統 – Proposal  

**文件版本**：v1.0  
**日期**：2026‑05‑10  
**專案代號**：Project 3

## 1. 專案概述  

| 項目 | 內容 |
|------|------|
| **系統名稱** | 小學二年級英語單詞學習系統 |
| **核心目標** | 以遊戲化、即時聲音回饋與獎勵機制提升二年級學生學習英語單詞的動機、興趣與記憶成效。 |
| **主要受益者** | 7‑8 歲小學二年級學生、家長、教師 |
| **商業價值** | 差異化教育產品、符合市場對兒童互動學習工具的需求、可作為後續付費擴充（進階關卡、教師版報表）之基礎平台。 |

## 2. MVP 範圍  

根據客戶確認，我們的 Minimum Viable Product (MVP) 包含以下 **核心功能**：

1. **單詞顯示與發音**  
   - 顯示約 30 個二年級常見英語單詞，搭配圖示與文字。  
   - 點擊單詞即播放正確發音音檔（支援本地化音檔快取）。  

2. **基本點擊配對遊戲**  
   - 文字‑圖像配對模式：玩家點選單詞與相應圖片，配對成功即顯示動畫與音效。  
   - 遊戲回合計時、即時正確/錯誤提示。  

3. **簡易積分與小物獎勵**  
   - 完成一輪配對即獲得積分。  
   - 積分可兌換「小物」貼紙（收集於個人背包）。  
   - 小物與積分顯示於遊戲介面右上角，配合簡易動畫增強正向回饋。  

4. **學習進度紀錄**  
   - 記錄每位學生已學、未學單詞、正確率與總積分。  
   - 家長/教師可查看個人儀表板（圖表式展示）以追蹤學習成效。  

5. **後台單詞與關卡管理**  
   - 文字、圖片、發音檔上傳與分類。  
   - 關卡（配對題組）設定，支援新增、刪除與排序。  
   - 積分與小物兌換規則配置。  

> **不在 MVP 範圍**：排行榜、成就徽章、進階遊戲模式、跨平台同步、付費功能、完整多語系支援。這些功能將於後續迭代中逐步加入。

## 3. 功能細節  

### 3.1 前端（React）  
| 功能 | 子項目 | 交互/UX 要點 |
|------|--------|---------------|
| 單詞卡片 | 圖示、文字、發音按鈕 | 色彩鮮明、按鈕尺寸符合 7‑8 歲手指操作，發音延遲 ≤ 150 ms |
| 配對遊戲 | 隨機排列的卡片、拖曳或點擊配對、成功動畫、錯誤音效 | 每次點擊即時回饋，動畫時間 ≤ 800 ms，避免卡頓 |
| 積分 & 小物 | 積分條、兌換按鈕、背包視圖 | 直觀圖示，音效提示「兌換成功」 |
| 進度儀表板 | 單詞掌握率圓形圖、積分歷史折線圖 | 提供簡易「家長模式」切換，隱藏遊戲 UI |
| 後台管理 | 單詞 CRUD、關卡設定表、兌換規則編輯 | 使用表單驗證、即時預覽，支援 CSV 匯入 |
| 音效管理 | 音量調節、靜音切換、音效預載 | 符合可訪問性需求，提供文字替代說明 |

### 3.2 後端（Node.js + Express）  
| API | 功能 | 安全/隱私 |
|-----|------|-----------|
| `/api/words` | 取得單詞清單、發音 URL、圖示 URL | 只讀，無個資 |
| `/api/game/result` | 上傳單輪遊戲結果（正確/錯誤、取得積分） | JWT 受保護（家長/教師帳號） |
| `/api/progress` | 讀寫學生學習紀錄（單詞掌握、積分） | 最小化資料、加密儲存 |
| `/api/rewards` | 兌換小物、取得背包內容 | 同上 |
| `/admin/*` | 後台 CRUD 操作（單詞、關卡、規則） | RBAC，僅限管理員 |
| `/api/auth` | 登入/註冊（家長、教師） | 符合 COPPA / GDPR‑Kids，需家長同意 |

## 4. 非功能需求  

| 項目 | 要求 | 測量方式 |
|------|------|-----------|
| **使用者體驗** | UI 色彩鮮明、操作簡潔、字體大小適合 7‑8 歲 | 用戶測試 - 目標 90% 正向回饋 |
| **互動延遲** | 點擊 → 音效 ≤ 200 ms，動畫啟動 ≤ 150 ms | 前端性能測試 |
| **可擴充性** | 後台可於 5 分鐘內批量上傳 100+ 單詞 | 後台測試 |
| **可維護性** | 前端採用 component‑based (React) + TypeScript；後端分層架構 | 代碼審查、CI 靜態分析 |
| **兼容性** | 支援 Chrome 80+, Safari 13+, Edge 最新版； iOS 12+, Android 8+ | 跨平台測試 |
| **安全與隱私** | 不收集個人身份資訊；如需帳號則採最小化資料、父母同意流程 | 合規審查 |
| **可訪問性** | 提供文字替代說明、音量控制、低對比模式 | WCAG AA 測試 |

## 5. 技術選型  

| 層級 | 技術 | 理由 |
|------|------|------|
| **前端框架** | **React + TypeScript** (Create‑React‑App / Vite) | 客戶已明確同意 React；社群成熟、元件化易於遊戲化 UI 開發 |
| **遊戲渲染** | HTML5 Canvas + PixiJS（或直接使用 React‑Spring 動畫） | 輕量、支援即時音效、跨平台 |
| **狀態管理** | Redux Toolkit (或 React Context) | 中小型專案足夠，便於跨頁共享積分與進度 |
| **後端框架** | **Node.js + Express** + TypeScript | 客戶同意 Node.js；快速開發 RESTful API |
| **資料庫** | PostgreSQL (雲端 RDS) | 結構化資料、支援複雜查詢與統計 |
| **認證與授權** | JWT + OAuth2 (可與家長 Google/Apple 登入整合) | 符合安全需求 |
| **雲端部署** | AWS (Elastic Beanstalk / ECS) 或 Azure App Service | 彈性、可自動擴展 |
| **音檔儲存** | Amazon S3 + CloudFront CDN | 低延遲、成本可控 |
| **CI/CD** | GitHub Actions + Docker | 自動化測試、部署一致性 |
| **測試框架** | Jest + React Testing Library (前端)；Mocha/Chai (後端) | 單元與端到端測試確保品質 |
| **分析/統計** | Google Analytics (只收集匿名使用行為) + 自建 KPI Dashboard | 追蹤使用時長、留存等指標 |

## 7. 成功指標 (KPIs)  

| KPI | 目標 | 量測方式 |
|-----|------|-----------|
| **每日使用時長** | ≥ 15 分鐘/生 | 後台使用紀錄 |
| **單詞掌握率** | 正確率 ≥ 85% | 系統測驗統計 |
| **活躍用戶** | 1‑月活躍率 ≥ 70% | 註冊帳號日活/月活比 |
| **家長/教師滿意度** | NPS ≥ 30 | 上線後第一季問卷 |
| **30 天留存** | ≥ 50% | 用戶留存曲線 |
| **獎勵兌換率** | 積分使用率 ≥ 60% | 後台兌換紀錄 |

## 8. 風險與因應措施  

| 風險 | 可能影響 | 緩解措施 |
|------|-----------|----------|
| **內容審核延遲**（單詞、圖像版權） | MVP 交付推遲 | 事前與教育顧問簽訂交付時程，使用開源或自行製作圖示 |
| **兒童隱私合規** | 法規風險、上線阻礙 | 采用最小化資料、家長同意工作流程，提前律師審核 |
| **性能瓶頸（音效延遲）** | 使用體驗下降 | 前端採用音效預載、CDN 加速；性能測試納入迭代驗收 |
| **跨平台兼容** | 部分設備卡頓 | 早期在 iOS/Android 模擬器與真機做兼容測試，必要時調整動畫帧率 |
| **開發人力波動** | 里程碑延誤 | 使用敏捷 Scrum，雙週回顧與資源彈性調整 |
