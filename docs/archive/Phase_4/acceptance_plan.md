# Phase 4 完整驗收計劃 (Acceptance Plan)

### 1. 前置確認 (Prerequisites)
請確認目前您的終端機（Terminal）中，後端與前端伺服器皆在運行狀態：
- **後端**：`uv run uvicorn backend.main:app` (Port 8000)
- **前端**：`npm run dev` (Port 5173)

---

### 2. 驗收步驟 (Testing Workflow)

#### 第一階段：認證與錯誤處理 (Auth & Error Handling)
1.  **登入失敗測試**：在登入頁輸入錯誤的密碼。
    - **預期結果**：出現紅色的 `Session expired or unauthorized` 錯誤提示，且**帳號與密碼欄位皆自動清空**。
2.  **正常登入**：使用 `admin` / `123456` 登入。
    - **預期結果**：順利進入 Dashboard 頁面。
3.  **登出功能**：點擊左下角頭像旁的「登出」按鈕。
    - **預期結果**：回到登入頁，且無法透過瀏覽器回退鍵進入 Dashboard。

#### 第二階段：專案管理 (Project Management)
1.  **建立專案**：點擊側邊欄的「+ 新建專案」。
    - **預期結果**：彈出玻璃擬態設計的模態框 (Modal)，輸入名稱後點擊建立。
2.  **側邊欄互動**：懸停在專案名稱上。
    - **預期結果**：出現刪除圖示，專案列表顯示「新建」標籤。

#### 第三階段：對話 UI 與輸入法 (Chat & IME Fix) — **關鍵測試**
1.  **中文字選字測試**：在輸入框輸入中文（例如「測試」）。
    - **測試動作**：按下 `Enter` 進行選字。
    - **預期結果**：**字體順利選取並顯示在框內，訊息「不會」被送出。**
2.  **換行功能測試**：輸入第一行字後，按下 `Shift + Enter`。
    - **預期結果**：輸入框自動增高，游標跳至下一行。
3.  **正式送出**：按下單獨的 `Enter` 鍵。
    - **預期結果**：訊息送出，輸入框清空並自動聚焦。
4.  **視覺組件**：觀察 Agent 回覆時是否有顯示：
    - **Typing Indicator** (Agent 思考中的三個動態圓點)。
    - **Phase Progress Bar** (上方五個階段的連線進度條)。

#### 第四階段：設定頁面 (Settings)
1.  **切換分頁**：進入左下角的「設定」圖示。
    - **預期結果**：可自由切換「LLM 供應商」與「SKILL 設定」。
2.  **表單設計**：觀察 API Key 欄位。
    - **預期結果**：預設為密碼隱藏狀態，可點擊眼睛圖示切換顯示。

#### 第五階段：檔案面板 (File Panel)
1.  **開啟面板**：點擊右上角的「文件」圖示。
    - **預期結果**：右側滑出玻璃質感的檔案面板。
2.  **檔案預覽**：點擊列表中的任何檔案（目前為 Mock 資料）。
    - **預期結果**：彈出全螢幕的 Markdown 預覽視窗。
